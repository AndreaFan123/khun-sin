# docs structure

| Folder | Contents |
|--------|----------|
| [product](product/) | Vision, requirements, roadmap, user research (e.g. project notes, report-system requirements) |
| [development](development/) | Architecture decision records (ADRs), technical specs, dev environment and deployment notes |
| [testing](testing/) | Test strategy, acceptance checklists (e.g. Lighthouse baseline, RWD checks) |
| [design](design/) | Design tokens, visual and interaction guidelines, data-viz styling |

Naming convention: ADRs are numbered sequentially (`ADR-001-*.md`, `ADR-002-*.md`, …) with status recorded inside each file (Proposed / Accepted / Deprecated / Superseded).
